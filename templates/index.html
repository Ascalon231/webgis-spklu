Pencipta & Pengembang Proyek

Cas.in (Charging Indonesia) adalah aplikasi WebGIS Portofolio Kasus Nyata (Real-Case Portfolio Project) yang dirancang untuk memetakan, memvisualisasikan, dan menganalisis sebaran Stasiun Pengisian Kendaraan Listrik Umum (SPKLU) di seluruh Indonesia. Proyek ini mengintegrasikan pemrosesan data spasial dengan antarmuka peta interaktif modern.

Arsitektur & Integrasi Alur Data Spasial

1. SHP / GeoJSON (Data Spasial Mentah) Raw Data

Data spasial SPKLU dari GIS Hub Kemenhub dan batas administratif BPS diolah serta disinkronkan ke format standar spasial WGS 84 (EPSG:4326).

2. Supabase + PostGIS (Spatial Database) DB & GIS Engine

Tempat penyimpanan tabel spasial `sebaran_spklu` dengan spatial index (GIST). Menjalankan kueri SQL spasial, deterministik Coordinate Jittering untuk koordinat bertumpuk, serta spatial intersect wilayah.

3. Flask RESTful API (Python Backend) API Backend

Menyediakan API endpoints aman (`/api/spklu`, `/api/stats`, `/health`), mengintegrasikan mesin perutean jalan (OSRM API), serta menangani keamanan API rate-limiting menggunakan Flask-Limiter.

4. Leaflet.js Client (Interactive WebGIS UI) Frontend UI

Visualisasi peta spasial interaktif bagi pengguna, marker clustering, buffer spasial 5km di sepanjang koridor rute, pencarian taksonomi terintegrasi, dan dashboard analitik grafis via Chart.js.

Fitur Teknik Rekayasa & GIS

PostgreSQL + PostGIS
Kueri spasial real-time berbasis serverless database Supabase (PostGIS cloud).
Jittering Spasial Otomatis
Mengurai titik koordinat tumpang-tindih secara otomatis di level database (SQL Window function).
Perutean Sepanjang Jalan
Pencarian SPKLU dalam radius koridor jalan nasional menggunakan buffer spasial 5km.
Keamanan API & Rate Limit
Mengamankan backend Flask dari penyalahgunaan menggunakan Flask-Limiter & monitoring /health.
Status Prototype & Kerja Sama Penerapan GIS

Proyek ini dikembangkan sebagai prototype portofolio aktif yang berjalan pada infrastruktur free tier (terbatas oleh limitasi database gratis Supabase, kuota API OSRM publik, rate limit server, dll.). Oleh karena itu, platform ini memiliki potensi besar untuk dideploy dan dikembangkan sepenuhnya bagi penggunaan publik skala besar maupun komersial industri EV.
